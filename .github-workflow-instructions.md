# GitHub Workflow Instructions

## üîÑ Always Check Repository Status First

Before making any changes to this project, **ALWAYS** check the GitHub repository status using these commands:

### 1. Check Git Status
```bash
git status --short
```
Shows which files have been modified but not committed.

### 2. Check Remote Connection
```bash
git remote -v
```
Confirms the GitHub repository is: `https://github.com/STaylor-Figma/User-Management-Portal-Geographies.git`

### 3. Check Recent Commits
```bash
git log --oneline -5
```
Shows the last 5 commits to see what's been pushed.

### 4. Pull Latest Changes
```bash
git pull origin main
```
Get the latest changes from GitHub before starting work.

---

## üìù Standard Commit & Push Workflow

After making changes:

```bash
# Stage your changes
git add <files>

# Commit with descriptive message
git commit -m "Brief description of changes"

# Push to GitHub
git push origin main
```
